  <name>Task 1: Install benchmark tooling and create benchmark script</name>
  <files>package.json, scripts/benchmark.mjs, lighthouse-reports/.gitkeep, .gitignore</files>
  <read_first>
    package.json
    .gitignore

  <action>
1. Install dev dependencies: `lighthouse` (latest), `chrome-launcher` (latest).
   Run: `npm install -D lighthouse chrome-launcher`

2. Create `lighthouse-reports/` directory with an empty `.gitkeep` file.

3. Add `lighthouse-reports/*.json` and `lighthouse-reports/*.html` to `.gitignore` (but keep the .gitkeep).
   Also add `dist/stats.html` to .gitignore.

4. Create `scripts/benchmark.mjs` -- a Node.js ESM script that:
   a. Uses `child_process.execFileSync` (NOT exec or execSync) to run `vite build` -- pass command as array `['vite', 'build']` with stdio: 'inherit'
   b. Spawns `vite preview --port 4173` using `child_process.spawn` (not exec) as a background process
   c. Waits 3 seconds for server ready (use a simple setTimeout promise)
   d. Launches Chrome via chrome-launcher with flags: `['--headless', '--disable-gpu', '--no-sandbox']`
   e. Runs Lighthouse on `http://localhost:4173` with options:
      - port from chrome instance
      - output: ['json', 'html']
      - onlyCategories: ['performance']
   f. Extracts from result.lhr:
      - categories.performance.score (multiply by 100)
      - audits['first-contentful-paint'].numericValue
      - audits['largest-contentful-paint'].numericValue
      - audits['total-blocking-time'].numericValue
      - audits['cumulative-layout-shift'].numericValue
      - audits['speed-index'].numericValue
   g. Writes results to `lighthouse-reports/baseline-{timestamp}.json` with structure:
      ```
      { timestamp, scenario: "initial-load", score, metrics: { fcp, lcp, tbt, cls, si } }
      ```
   h. Writes the HTML report to `lighthouse-reports/baseline-{timestamp}.html`
   i. Prints a summary table to console (score, FCP, LCP, TBT, CLS, SI with units)
   j. Kills Chrome and the preview server process in a finally block
   k. Exits with code 1 if performance score < 50 (indicating serious issues)

5. Add npm scripts to package.json:
   - `"benchmark": "node scripts/benchmark.mjs"`
   - `"build:analyze": "vite build"` (will produce stats.html via visualizer configured in task 2)

  IMPORTANT: Use `execFileSync` and `spawn` instead of `exec`/`execSync` to avoid shell injection. The project has a security rule requiring `execFile` over `exec`.
  The script should handle errors gracefully: if chrome-launcher fails, print a message suggesting the user install Chrome/Chromium.
  </action>
