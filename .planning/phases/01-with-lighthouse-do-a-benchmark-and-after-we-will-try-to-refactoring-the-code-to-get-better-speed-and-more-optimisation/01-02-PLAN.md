1. Read the current vite.config.ts and understand all existing configuration.

2. Add `build.rollupOptions.output.manualChunks` to vite.config.ts with the following chunking strategy:

```typescript
build: {
  commonjsOptions: {
    transformMixedEsModules: true,
  },
  rollupOptions: {
    output: {
      manualChunks(id) {
        if (id.includes('node_modules')) {
          // React core - stable, cacheable
          if (id.includes('/react/') || id.includes('/react-dom/')) {
            return 'react-vendor';
          }
          // CodeMirror editor - large but stable
          if (id.includes('@codemirror/') || id.includes('codemirror') || id.includes('@lezer/')) {
            return 'codemirror-vendor';
          }
          // Mermaid is already auto-split by diagram type by Vite's default chunking.
          // Keep the mermaid core + elk layout in its own chunk.
          if (id.includes('mermaid') && !id.includes('mermaid-studio')) {
            // Let Vite's default chunking handle mermaid - it already splits diagram types
            return undefined; // returns undefined means let default chunking handle it
          }
          // Lucide icons
          if (id.includes('lucide-react')) {
            return 'lucide';
          }
          // i18n
          if (id.includes('i18next') || id.includes('react-i18next')) {
            return 'i18n';
          }
          // Other vendor
          return 'vendor';
        }
      },
    },
  },
},
```

CRITICAL: Preserve ALL existing config:
- Keep `plugins: [react(), visualizer({...})]`
- Keep `resolve: { tsconfigPaths: true }`
- Keep `optimizeDeps: { exclude: [...], include: [...] }`
- Keep `server: { port: 5173, strictPort: false }`
- Keep `build.commonjsOptions: { transformMixedEsModules: true }` -- move into the same build block

3. After updating, run `npx vite build` and verify:
   - The "Some chunks are larger than 500 kB" warning should be reduced or eliminated for the main chunks
   - Check that the output now has separate `react-vendor`, `codemirror-vendor`, `lucide`, `i18n`, and `vendor` chunks
   - Total gzip size should be similar (just distributed differently)
   - The mermaid diagram-type chunks (flowDiagram, sequenceDiagram, etc.) should remain auto-split

4. If the `render-*.js` chunk is still > 500KB after splitting out the above, it means mermaid core itself is too large. In that case, do NOT try to split mermaid further (it already auto-splits diagram types and splitting its core would break rendering). Accept that mermaid is a large dependency and the optimization comes from better caching strategy, not smaller mermaid.

5. Verify the app still works: run `npx vite preview` briefly to check the build is valid.
